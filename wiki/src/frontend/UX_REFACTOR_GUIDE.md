# [Codex/AI용] Olfit 이미지 업로드 및 분석 프로세스 개선 가이드

> **⚠️ 필수 준수 사항 (Strict Permission & Scope Constraint)**
> - 본 가이드를 수행하는 AI 및 개발자는 **Frontend 관련 코드(`frontend/` 디렉토리 내)**만 수정할 권한이 있습니다.
> - Backend, Models, Database 등 Frontend 외부의 코드는 절대 수정하거나 접근해서는 안 됩니다.
> - 오직 요청받은 특정 파일들만 수정하며, 그 외의 파일에 영향을 주지 않도록 주의하십시오.

이 문서는 Olfit 프로젝트의 이미지 업로드 후 즉시 분석되는 로직을 개선하여, 사용자가 확인 후 분석을 시작하거나 이미지를 초기화할 수 있는 기능을 추가하고, 리포트 하단에서 재분석을 진행할 수 있는 버튼을 추가하기 위한 가이드입니다.

## 1. 관련 파일 주요 내용 정리 (Summary)

### 📂 `frontend/src/components/common/ImageUploader.tsx`
*   **역할**: OOTD 이미지를 업로드, 리사이징, S3 업로드 시뮬레이션을 담당하는 컴포넌트입니다.
*   **현재 로직**: `processImage` 함수 내에서 S3 업로드(`uploadToCloudStorage`)가 완료되면 즉시 `onImageProcessed` 콜백을 호출하여 상위 컴포넌트(`AIInterviewSection`)의 분석 로직을 트리거합니다.
*   **수정 포인트**:
    *   이미지 업로드 완료 후 바로 `onImageProcessed`를 호출하지 않고, "분석 대기" 상태를 유지해야 합니다.
    *   미리보기(`preview`)가 존재하고 분석 중(`isAnalyzing`)이 아닐 때, **[분석 시작]** 버튼과 **[초기화]** 버튼을 표시해야 합니다.

### 📂 `frontend/src/components/sections/AIInterviewSection.tsx`
*   **역할**: 이미지 업로더를 포함하며, 실제 AI 분석 API 호출 및 로딩 애니메이션(프로그레스 바)을 제어합니다.
*   **현재 로직**: `handleImageProcessed`가 호출되면 `isLoading` 상태를 `true`로 만들고 5초간의 분석 시뮬레이션을 시작합니다.
*   **수정 포인트**:
    *   이미지 업로더에서 전달된 `onImageProcessed` 호출 시에만 분석 시뮬레이션이 시작되도록 유지하되, 전체적인 UX 흐름이 "업로드 완료 -> 사용자 확인 -> 분석 시작"이 되도록 연동을 확인합니다.

### 📂 `frontend/src/components/report/RecommendationList.tsx`
*   **역할**: 분석 결과 리포트의 맨 하단에 위치하며, 향수 추천 리스트와 매칭 이유를 보여줍니다.
*   **현재 로직**: 추천 상품 카루셀과 정렬 버튼만 존재합니다.
*   **수정 포인트**:
    *   컴포넌트 하단에 **[분석 리포트 다시해보기]** 버튼을 추가합니다.
    *   이 버튼 클릭 시 전역 상태(`useOlfitStore`)를 초기화하고 인터뷰 섹션으로 스크롤 이동시키는 로직이 필요합니다.

## 2. 실무 및 QA 관점의 고려 사항 (Professional & QA Focus)

실무에서 QA팀이나 사용자에게 테스트를 맡기기 전, 아래와 같은 전문적인 품질 기준을 반영하여 수정해야 합니다.

### 🛡️ 예외 처리 및 안정성 (Error Handling & Stability)
- **API 실패 대응**: 이미지 업로드 중 네트워크 오류나 서버 에러(500) 발생 시, 사용자에게 명확한 에러 메시지를 보여주고 '다시 시도'할 수 있는 경로를 제공해야 합니다.
- **이미지 검증**: 프론트엔드 단에서 파일 크기(예: 10MB 이하), 확장자(JPG, PNG, WEBP)를 엄격히 체크하여 서버 부하를 방지해야 합니다.

### 🎨 UX 피드백 및 상태 관리 (UX Feedback & State)
- **버튼 상태 제어**: 분석이 시작되면 버튼을 `disabled` 처리하여 중복 클릭으로 인한 API 중복 호출을 원천 차단해야 합니다.
- **로딩 상태의 시각화**: 단순히 멈춰있는 것이 아니라, 스켈레톤 UI나 로딩 스피너를 통해 "작업이 진행 중임"을 명확히 전달해야 합니다.
- **초기화(Reset)의 완전성**: '다시해보기' 클릭 시 이전 분석 결과가 메모리에 남아있지 않도록 상태를 완전히 비워야 합니다.

### 📱 반응형 및 접근성 (Responsive & a11y)
- **다양한 해상도 대응**: 모바일 환경(작은 화면)에서도 버튼이 겹치지 않고 터치하기 적절한 크기(최소 44px)를 유지해야 합니다.
- **접근성**: 이미지 업로드 및 버튼 요소에 적절한 `aria-label`을 부여하여 스크린 리더 사용자도 배려해야 합니다.

### 🚀 성능 및 최적화 (Performance)
- **불필요한 리렌더링 방지**: 분석 결과와 관계없는 UI 요소들이 분석 중에 불필요하게 다시 그려지지 않도록 `React.memo`나 `useCallback`을 적절히 사용해야 합니다.

## 3. 수정을 위한 구체적인 프롬프트 (Modification Prompt)

```markdown
### [Task] 이미지 업로드 확인 단계 추가 및 리포트 하단 재분석 버튼 구현 (QA-Ready)

**목표**: 실무 수준의 안정성과 UX를 갖춘 이미지 분석 프로세스를 구현합니다.

**요구 사항**:

1. **ImageUploader.tsx 수정**:
   - `processImage` 내부에서 S3 업로드 완료 후 `onImageProcessed`를 즉시 호출하는 로직을 제거하세요.
   - 업로드 완료 여부를 관리하는 상태(`isReady`)를 추가하고, 이미지 프리뷰 아래에 버튼 영역을 추가하세요.

   - **[초기화]** 버튼: 클릭 시 현재 미리보기를 제거하고 업로드 전 상태로 되돌립니다.

2. **RecommendationList.tsx 수정**:

3. **안정성 및 성능**:
   - 모든 신규 버튼에 `aria-label`을 추가하세요.
   - `RecommendationList` 하단의 재시작 버튼 클릭 시 `window.scrollTo` 대신 `element.scrollIntoView({ behavior: 'smooth' })`를 사용하여 UX를 개선하세요.

4. **제약 사항**:

```
